## ⚙️ **How SQL Transformations work**

At a high level, SQL Transformations lets you:

- **✏️ Manipulate and transform data using SQL queries**
- **📈 Perform powerful data operations on large datasets — quickly and efficiently**

The workflow decomposes into three configuration areas: **data source setup**, **query setup**, and **output configuration**.

---

### 🗂️ Data source setup

> 📌 SQL Transformations supports **three data source types**:
> 
> - **📄 CSV files**
> - **📁 Workato FileStorage** — Workato's built-in persistent file storage
> - **📊 Data Tables** — Workato's structured data storage

This flexibility is why SQL Transformations can span so many sources: they get **normalized through CSV / FileStorage / Data Tables** before the transformation runs.

---

### 🔎 Query setup

Standard SQL operations are supported:

- **🔗 Joins**
- **📊 Aggregations**
- **🔍 Filtering**
- **↕️ Sorting**
- **➕ And more**

---

### 📤 Output configuration

> 📌 Two output modes:
> 
> - **🌊 Content stream** — output the results directly downstream as a data stream.
> - **📁 FileStorage file** — write results to a persistent file in Workato FileStorage.

---

## 🧩 **Problems SQL Transformations solve**

SQL Transformations exists to solve a specific set of large-scale data problems:

- **📈 Handling large volumes of data** — millions of rows in a single operation.
- **🔄 Handling complex data transformations** — multi-source joins, aggregations, transformations.
- **⚡ Performing at scale** — bulk operations, not row-by-row loops.
- **🧹 Data validation and cleansing** — deduplicate, error-check, sanitize.
- **🔎 Change Data Capture (CDC)** — identify changes over time, synchronize data across systems, capture and handle changes.
- **✨ Data enrichment** — merge data from multiple sources.
- **🚀 Optimizing performance** — replace slow, looped, single-record patterns with bulk transforms.

---

## 🚀 **Module key takeaways**

- **Three data sources**: CSV files, Workato FileStorage, Data Tables — normalized before transformation.
- **Standard SQL operations**: joins, aggregations, filtering, sorting, more.
- **Two output modes**: content stream (downstream) or FileStorage file (persistent).
- **Workato FileStorage is central** — it's the staging area for large datasets before and after transformation.
- **Bulk-first**: SQL Transformations processes millions of rows in a single operation, replacing slow row-by-row patterns.
