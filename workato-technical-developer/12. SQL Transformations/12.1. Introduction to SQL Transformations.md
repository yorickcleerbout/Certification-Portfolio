## 🔧 **What is SQL Transformations?**

> 📌 **SQL Transformations** is a Workato tool for handling **complex, large-scale data transformations** using **SQL (Structured Query Language)**. It lets you efficiently manipulate and transform data extracted from many sources — enabling seamless data integration and orchestration.

Two headline capabilities:

- **⚙️ Perform complex data transformations** at scale
- **✏️ Manipulate and modify data** using standard SQL

> 💡 **Not to be confused with SQL Collection (chapter 6)** — the two look similar but optimize for different scenarios. SQL Transformations is for **persistent, large-scale, cross-job** work; SQL Collection is for **temporary, single-job, in-memory** work. Full comparison in 12.3.

---

## ⭐ **Key features**

Four defining features of SQL Transformations:

- **🌐 Query data from any number of sources**
- **📈 Virtually no limit on volume**
- **⚡ High-performance query execution**
- **🧮 Support for complex functions**

---

## 🎯 **Main purpose**

> 📌 The primary purpose is to **process and transform large datasets quickly and effectively** — especially when **integrating data from multiple sources** and **loading it into a target system**.

Classic ETL/ELT territory: pull from many places, transform in bulk, deliver to one destination.

---

## 🔄 **How SQL Transformations are used**

The four-step usage flow:

1. **🔌 Connect your data sources**
2. **📝 Define the SQL query**
3. **▶️ Execute the query**
4. **📤 Configure the output**

Details on how each step is set up are covered in 12.2 and 12.3.

---

## 💡 **Use cases**

Two illustrative scenarios showing how SQL Transformations fit into real workflows.

---

### 🔎 Use case 1: Change Data Capture (CDC)

Identify and extract **new or updated data** from a source system — critical for keeping downstream systems in sync without re-processing everything.

> **Problem:** Fetching only new or updated data.
> 
> **Solution:** Use SQL Transformations to compare historical data with new data and extract only the differences.

The workflow:

1. **⚙️ Configure the source system** to look for new data.
2. **🔎 Use the Query CSV action** to compare historical data with new data.
3. **📤 Extract the changed records** and send them to cloud storage (e.g. **Amazon S3**).

---

### ✨ Use case 2: Data enrichment

Enrich data with additional information **from multiple sources** before sending it onward.

> **Problem:** Enriching data with additional info from multiple sources.
> 
> **Solution:** Use SQL Transformations to merge data from multiple sources and enrich the target dataset.

The workflow:

1. **🕒 Scheduler trigger** to fetch data from **Salesforce** in bulk.
2. **📥 Fetch additional info** from other sources.
3. **🔀 Use the Query CSV action** to merge and enrich the data.
4. **📤 Send the enriched data** to an **SFTP server** or any other destination.

## 🚀 **Module key takeaways**

- **SQL Transformations = large-scale SQL-based data transformation** built into Workato — no external DB required.
- **Four key features**: query many sources, virtually no volume limit, high performance, complex functions.
- **Four-step flow**: connect sources → define query → execute → configure output.
- **Two canonical use cases**: **CDC** (extract diffs by comparing historical vs new) and **data enrichment** (merge multi-source data).
- **The Query CSV action** is the workhorse for both use cases — remember it.
