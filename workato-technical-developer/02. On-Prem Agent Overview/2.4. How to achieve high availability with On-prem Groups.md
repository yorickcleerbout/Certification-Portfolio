## 🏗️ **On-Prem Groups**

> 📌 A **Workato On-Prem Group** is a **logical grouping of one or more on-prem agents into a common connection pool**. Workload is distributed **evenly across all agents** in the group.

Grouping agents this way unlocks three operational benefits — high availability, fault tolerance, and load balancing.

---

### 🛡️ High availability

> 📌 A recipe using an on-prem group with **more than one active agent** achieves **High Availability**. Multiple agents serve a single recipe connection, so if one OPA goes down, the others keep operating — the recipe doesn't go offline with the single failing node.

---

### 🔄 Fault tolerance

**Fault tolerance** is a system's ability to continue operating despite the failure of some components.

In on-prem groups, fault tolerance is achieved by adding **redundancy** — each agent acts as a backup or fail-safe for the others. If one or more hosting servers goes offline, the remaining agents pick up the work.

---

### ⚖️ Load balancing

**Load balancing** is the distribution of work across multiple resources. For on-prem groups, that means requests are spread evenly across the agents in the group. This:

- Avoids overloading a single server.
- Improves responsiveness.
- Improves overall performance of recipes using on-prem connections.

---

### 🧠 Quick recall

- An On-Prem Group is a logical grouping of agents into a `_____`. (Common connection pool)
- How is workload distributed across agents in a group? (Evenly across all agents.)
- How many active agents are needed in a group to achieve High Availability? (`_____`) (More than one)
- Name the three operational benefits of on-prem groups. (High availability, fault tolerance, load balancing)
- What's the difference between high availability and fault tolerance? (HA = service stays up when an agent fails; fault tolerance = the broader principle of redundancy that makes HA possible. Both rely on having multiple agents.)
- A single agent goes offline. What happens to a recipe using the group? (The remaining agents keep serving the connection — no interruption.)

---

## 🚀 **Module key takeaways**

- An **On-Prem Group** is a logical pool of one or more agents — Workato spreads workload **evenly** across them.
- Grouping enables three things at once: **High Availability** (service stays up when an agent fails), **Fault Tolerance** (redundancy backing it up), and **Load Balancing** (work distributed across agents).
- The minimum for HA is **more than one active agent** — a single-agent group is just an agent.
