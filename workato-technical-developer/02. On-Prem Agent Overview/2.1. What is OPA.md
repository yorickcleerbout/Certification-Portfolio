## 🏢 **What is "On-Prem"?**

> 📌 **OPA = On-Prem Agent**

Historically, **"on-prem"** (short for _on-premise_) referred exclusively to software deployed onto infrastructure **fully controlled by an organization** — physical servers and datacenters.

More recently, the term has loosened. Today, **on-prem can also refer to software deployed onto cloud Infrastructure-as-a-Service** such as **AWS, Azure, and GCP** — as long as the customer controls the infrastructure.

---

## 🌐 **What is the Workato On-Prem Agent (OPA)?**

> 📌 The **Workato On-Prem Agent** is a lightweight **Java program** you install on a protected server inside your company's network. It provides a **secure way for Workato to selectively access customer-authorized on-prem apps** — legacy systems, databases, folders — that aren't reachable on the public internet.

The key benefit:

> ✅ **No need to open ports in your corporate firewall.** The agent is installed and runs inside your network, alongside the systems you want to connect to, and Workato communicates with it through an outbound channel (covered in 2.3).
