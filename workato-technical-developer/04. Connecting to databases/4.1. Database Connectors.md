## 🗄️ **What is a database?**

> 📌 A **database** is a collection of data stored electronically for data retrieval and transformation. Whether it's rows in a table or arrays of values, it's a database if the goal is structured retrieval and manipulation.

Databases are everywhere because they're a convenient format for handling large sets of information — especially for data transformation and analysis. Common reasons to use them:

1. **📦 Store large amounts of information** — far more than you'd reasonably keep in memory or flat files.
2. **🔍 Select certain data or identify patterns** — query specific slices instead of working with everything at once.
3.  🛠️**Manipulate data** to collect coherent information.

---

## 🔌 **Connecting databases in Workato**

Workato lets you connect databases from various sources for data manipulation. **Two criteria** determine which connector to use:

- **☁️ Cloud vs on-prem** — where the database lives.
- **📦 Workato OOB vs JDBC** — whether a pre-built Workato connector exists for that database.

---

### ☁️ Cloud vs on-prem

Connect databases either through **cloud** applications, or via **on-premise (OPA)** connectivity if your database network doesn't allow direct connection from the internet.

---

### 📦 Workato OOB vs JDBC connectors

|Connector type|When to use|
|---|---|
|**🧱 Workato OOB (Out-Of-the-Box) connectors**|Pre-built Workato connectors — native apps and external app integrations (e.g. _Variables by Workato_, _SQL Server_). Always check OOB first; using pre-established connectors streamlines integration.|
|**🔗 JDBC (Java Database Connectivity) connectors**|Use when the database has **no existing Workato OOB connector**. JDBC is an API that lets Java applications connect to ANSI-compliant databases (Snowflake, SAP HANA, etc.).|

> 📌 **JDBC requires deploying Workato OPA (On-Prem Agent)** to download a JDBC driver. OPA is a secure way for Workato to selectively access customer-authorized databases.

---

### 🧭 Choosing your database connection

Combine both criteria — _where is the database?_ and _is there an OOB connector?_ — to pick the right path:

|If the database is…|Workato OOB connector|JDBC connector|
|---|---|---|
|**☁️ Cloud**|Find a compatible Workato app connector to connect on cloud. Requests execute from Workato servers.|**❌ Not possible**|
|**🏢 On-prem**|Find a compatible Workato app connector to connect on-prem. Requests are sent from the environment where OPA is installed.|If no compatible OOB connector exists, use Workato OPA to download a JDBC driver.|

> ⚠️ **The single most important constraint:** JDBC connectors **cannot be used for cloud databases**. JDBC requires OPA, which is on-prem only. For a cloud database with no OOB connector, you're out of luck with JDBC.

---

### 🧠 Quick recall

- Name the two criteria for choosing a database connection in Workato. (Cloud vs on-prem; Workato OOB vs JDBC)
- Can you use a JDBC connector for a cloud database? (No — JDBC requires OPA, which is on-prem only.)
- OPA stands for `_____`. (On-Prem Agent)
- Name an OOB native Workato app used for database integration. (Variables by Workato; also SQL Server is OOB but external.)
- When should you reach for JDBC? (When no Workato OOB connector exists for the database, AND the database is on-prem.)
- Which two databases are mentioned as ANSI-compliant examples that use JDBC? (Snowflake, SAP HANA)

---

## 🚀 **Module key takeaways**

- A **database** is a structured collection of data optimized for retrieval and transformation.
- **Two criteria** to pick a database connection in Workato: location (**cloud vs on-prem**) and connector type (**OOB vs JDBC**).
- **Always try OOB first** — it's the fastest, most reliable path. JDBC is the fallback for unsupported databases.
- **JDBC + cloud = impossible.** JDBC requires the **On-Prem Agent (OPA)**.
- _Variables by Workato_ is a native OOB connector you can use directly inside recipes.
