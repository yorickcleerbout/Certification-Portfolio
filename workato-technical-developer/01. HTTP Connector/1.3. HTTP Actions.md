## ⚡ **What is an HTTP action?**

> 📌 **HTTP actions** enable you to create custom requests and response schemas for HTTP-based APIs.

Once your HTTP connection is set up, you can configure HTTP actions. The HTTP Connector exposes two action types:

- **📤 Send request**
- **⏳ Send request and wait for response** (private preview)

---

### ⚖️ Send request vs Send request and wait for response

| |📤 **Send request**|⏳ **Send request and wait for response**|
|---|---|---|
|**Request timeout**|Up to **120 seconds**|Up to **1 hour**|
|**Response size**|Soft limit up to **200 MB**|Hard limit of **100 MB**|
|**Connection types**|All HTTP connection types supported|**Custom SSL certificates and OPA NOT supported**|
|**Job processing**|Jobs in queue processed **in order**|Jobs may not be processed in order|
|**Setup**|Can use the HTTP wizard|No HTTP wizard (start with Send request, then switch)|
|**Availability**|Generally available|Private preview — contact your Workato rep|

> 📌 **Recommendation:** start with **Send request** when building. If timeouts occur, switching to **Send request and wait for response** is easy after the initial recipe is in place.

> ⚠️ Two limits go in _opposite_ directions between these actions: response size is **larger** for Send request (200 MB soft) but timeout is **shorter** (120s). The wait-for-response variant flips this: **smaller** response cap (100 MB hard) but **much longer** timeout (1 hr). Pick based on the constraint that hits first.

## 🧙‍♂️ **The Workato HTTP wizard**

The HTTP Connector includes a **3-step wizard** that guides users through building HTTP actions, with built-in testing functionality.

Key features:

- **🤖 ML-based suggestions** — recommends a request body schema based on the API endpoint URL.
- **🧪 Test console** — instant testing and validation of HTTP requests.
- **🔍 Full request/response visibility** — returns the complete API request and response for troubleshooting.
- **📦 Auto-generated response body and headers** — Workato builds these for you.

The course uses the wizard in labs. Once you're comfortable with it, you can also configure HTTP connections and actions manually — see the Workato documentation for details.

## ⚠️ **Error handling**

Error messages are critical for identifying the root cause of a failed action.

> 📌 Whenever the response from a **Send request** action contains a **non-2XX response code**, the error message in the body is returned.

You can view error messages and response headers in your **recipe job report**. As a best practice, add an **error monitoring block** to every recipe. Specific behaviors:

- **🔁 Rate limit errors** — configure the recipe to retry the HTTP action after a specified wait.
- **📧 Error email notifications** — the HTTP error message is automatically included.
- **📝 Logging** — the HTTP error message can be added to logs with **Activity log stream**.

## 🚀 **Module key takeaways**

- **Two HTTP action types**: **Send request** (GA, 120s timeout, 200 MB soft, all connections, in-order) vs **Send request and wait for response** (private preview, 1 hr timeout, 100 MB hard, no custom SSL/OPA, may not be in order).
- **Start with Send request** by default; switch to the wait-for-response variant only when timeouts demand it.
- The **HTTP wizard** is a 3-step guided builder with ML schema suggestions, a test console, and full request/response visibility for troubleshooting.
- For errors: non-2XX responses surface the error message body; view it in the job report; add error monitoring blocks for retry, email alerts, and Activity log stream logging.
