## 📡 **What is an API?**

> 📌 **API** stands for **Application Programming Interface** — a mechanism that lets two applications exchange information using a set of rules.

When end users interact with an application, they go through a user interface. APIs play the same role between _applications_ — they provide the ability to **interact with applications from within other applications**.

---

## 🧬 **Anatomy of a REST API**

A typical REST API has **seven** components. Understanding each is the foundation for designing or consuming APIs.

---

### 🎯 1. Endpoint

The **endpoint** is the address you send your request to. It's the most important part of an API because it's what developers implement to make requests.

```
https://app.workato.com/workato-webhook/api/
```

---

### 📦 2. Resource

The **resource** is the type of info the API is tagged with — the object or record the endpoint interacts with.

```
/api/users
```

---

### 🔧 3. Method

The **method** defines the operation on the resource. The four most common:

|Method|What it does|
|---|---|
|**`GET`**|Get a resource from the server|
|**`POST`**|Create a resource on the server|
|**`PATCH`** / **`PUT`**|Update an existing resource on the server|
|**`DELETE`**|Delete an existing resource from the server|

---

### 🔗 4. Request body / query params

Two ways to pass data into an API call:

- **Query parameters** — used for `GET` requests. Appear in the URL after `?`, each pair separated by `&`:
    
    ```
    ?query1=value1&query2=value2
    ```
    
- **Request body** — used for `POST` / `PATCH` / `PUT` requests. Typically JSON:
    
    ```json
    {  "user": {    "name": "User1",    "Role": "Operator"  }}
    ```
    

---

### 📋 5. Headers

**Headers** are additional metadata for communication between client and server. Two you'll see most often:

|Header|Purpose|
|---|---|
|**`Authorization`**|Contains the authentication credentials for HTTP authentication.|
|**`Content-Type`**|Tells the client what media type (e.g. `application/json`) the response is sent in — important so the client knows how to process the response body.|

---

### 📩 6. Response body

The **response body** contains the resource data the client requested — and additional output ports. Some APIs are configured to return errors in their response body too.

```json
{
  "name": "Madison Bumgarner",
  "email": "ace@rightscale.com",
  "company": "ABC Aces"
}
```

---

### 🚦 7. HTTP status codes

> 📌 A status code is a **three-digit integer**. The first digit identifies the response category.

|Range|Meaning|
|---|---|
|**1xx**|Informational message only|
|**2xx**|Success|
|**3xx**|Redirect to another URL|
|**4xx**|Client-side error|
|**5xx**|Server-side error|

---

### 🧠 Quick recall

- API stands for `_____`. (Application Programming Interface)
- According to the source, which API component is the _most important_? (Endpoint — it's what developers implement to make requests.)
- Which HTTP method creates a new resource? (POST)
- Which HTTP method(s) update an existing resource? (PATCH or PUT)
- Query params go with `_____` requests; request bodies go with `_____` requests. (GET; POST/PATCH/PUT)
- A 200 falls in the `_____` category; a 404 in the `_____`; a 503 in the `_____`. (2xx success; 4xx client error; 5xx server error)
- Which header tells the client what media type the response is in? (Content-Type)

## 💼 **Why expose an API?**

Four business reasons to expose APIs:

- **💰 New revenue channels** — leverage data flow from apps and consumers to build intelligent, personalized experiences. A public API program can inspire an ecosystem of app developers creating new experiences and breakthrough apps.
- **🔐 Secure data sharing** — expose hidden data to end users in a secure, reliable manner. Share data with external partners and customers in a controlled way.
- **⚡ Real time** — migrate from legacy batch integration processes to modern real-time patterns, enabling modern automation use cases.
- **📜 Compliance** — enable interoperability standards for sharing information between systems. Regulators can require organizations to expose data as services.

---

### 🧠 Quick recall

- Name the four reasons to expose an API. (New revenue channels; secure data sharing; real-time patterns; compliance)
- What does exposing an API let you migrate _away from_? (Legacy batch integration — to real-time patterns instead.)
- Why does compliance show up as a reason? (Regulators can demand organizations expose data as services for interoperability.)

## 🛠️ **API Platform features (from the knowledge check)**

The knowledge check below tests three Workato API Platform features that aren't elaborated in the lesson body. Worth surfacing here for studying:

> 📌 **📊 Dashboard** — allows API platform owners to **visualize real-time data** pertaining to endpoints and API collections at a glance.

> 📌 **📚 Library** — a **centralized catalog of API collections** that are **discoverable by other users** in your organization.

> 📌 **🚪 Access Policies** — control **access** to APIs, but **do NOT control client usage** (rate limits, quotas). For usage control, look to API policies instead.

## 🚀 **Module key takeaways**

- **API** = Application Programming Interface. APIs let applications talk to each other the same way UIs let users talk to applications.
- A REST API has **seven anatomy parts**: endpoint, resource, method, request body/query params, headers, response body, status codes.
- **Methods**: GET = read, POST = create, PATCH/PUT = update, DELETE = remove.
- **Status codes**: 1xx info, **2xx success**, 3xx redirect, **4xx client error, 5xx server error**.
- **Four reasons to expose APIs**: revenue, secure sharing, real-time, compliance.
- **API Platform key features**: Dashboard (visualize), Library (discover), Access Policies (control _access_, not usage).
